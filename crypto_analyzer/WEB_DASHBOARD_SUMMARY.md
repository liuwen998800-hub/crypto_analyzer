### 🚀 技术架构

#### 前端技术栈
- **HTML5/CSS3**: 现代化语义化标签，CSS Grid/Flexbox布局
- **JavaScript ES6+**: 异步数据加载，动态DOM操作
- **Bootstrap 5**: 响应式框架，组件库
- **Chart.js 3.x**: 数据可视化图表库
- **Font Awesome**: 图标系统

#### 后端技术栈
- **Flask**: 轻量级Python Web框架
- **REST API**: 标准接口设计
- **JSON**: 数据交换格式
- **文件系统**: 结果持久化存储

#### 数据流架构
```
[分析系统] → [JSON文件] → [Flask API] → [前端界面] → [用户]
    ↓            ↓            ↓            ↓          ↓
Python脚本   results/    REST接口    HTML/JS     交互操作
            latest.json  数据代理    Chart.js    数据可视化
```

### 🛠️ 使用指南

#### 快速启动
```bash
# 1. 进入项目目录
cd crypto_analyzer

# 2. 启动Web仪表板
cd web_dashboard
chmod +x start.sh
./start.sh

# 3. 打开浏览器访问
# http://localhost:5000
```

#### 主要操作
1. **查看市场总结** - 首页顶部面板
2. **查看币种分析** - 中间卡片区域
3. **查看价格图表** - 底部交互图表
4. **查看系统状态** - 右侧边栏
5. **查看币种详情** - 点击任意币种卡片
6. **重新运行分析** - 点击导航栏"重新分析"按钮
7. **导出数据** - 点击右侧"导出数据"按钮

#### API接口使用
```bash
# 获取最新分析结果
curl http://localhost:5000/api/latest

# 获取市场总结
curl http://localhost:5000/api/market_summary

# 获取BTC详情
curl http://localhost:5000/api/coin/BTC

# 运行新的分析
curl http://localhost:5000/api/run_analysis

# 获取系统状态
curl http://localhost:5000/api/status
```

### 🔒 安全考虑

#### 生产部署建议
1. **禁用调试模式**: 设置 `debug=False`
2. **使用HTTPS**: 配置SSL证书
3. **访问控制**: 设置IP白名单或登录验证
4. **API限流**: 限制接口调用频率
5. **日志监控**: 记录访问日志和错误日志

#### 数据安全
1. **API密钥**: 不在前端暴露敏感信息
2. **输入验证**: 所有API参数都验证过滤
3. **文件权限**: 结果文件设置适当权限
4. **备份策略**: 定期备份分析结果

### 📋 部署选项

#### 开发环境
```bash
# 直接运行
python3 app.py

# 或使用脚本
./start.sh
```

#### 生产环境
```bash
# 使用Gunicorn
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 app:app

# 使用systemd服务
sudo systemctl start crypto-analyzer-web
```

#### Docker部署
```dockerfile
FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:5000", "app:app"]
```
